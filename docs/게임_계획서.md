# 노가다 MASTER — 게임 계획서

> **개발·기획용** 통합 문서. 현재 구현 상태와 로드맵을 담습니다.

---

## 1. 게임 개요

### 1.1 컨셉 (3줄)

- **노가다 MASTER**: 평민으로 시작해 **채집·낚시·채광·장사**로 아이템을 모으고, 레벨·칭호·골드를 쌓아 **마스터**까지 성장하는 웹 육성 게임.
- **에너지 기반 루프**: 행동 1회당 에너지 1~2 소모. 일일 리셋으로 반복 플레이 유도.
- **4직업 공통**: 농부·낚시꾼·광부·장사꾼 중 랜덤 직업. 직업별 드랍·칭호·장사 루트가 다르지만, 모든 활동은 공통 가능.

### 1.2 목표 플레이어 경험

- **시작**: 아이디·비밀번호 → 직업 랜덤 배정 → 채널 입장.
- **메인**: 지도에서 **밭 / 광장 / 장터** 이동 → 밭에서 직업별 행동(채집·낚시·채광·장사) 수행.
- **성장**: 아이템 획득 → 가방·조합·장터 판매 → 골드·경험치·칭호 달성.
- **엔드**: 마스터 칭호 달성 후에도 무한 성장. 명시적 게임오버 없음.

## 2. 현재 구현 상태

### 2.1 완료된 기능

| 영역 | 기능 | 비고 |
|------|------|------|
| **캐릭터** | 아이디(2~12자)·비밀번호, 직업 랜덤, 채널 입장 | 농부/낚시꾼/광부/장사꾼 |
| **에너지** | 일일 리셋, 행동당 소모 | 최대 100 (레벨 의존) |
| **행동** | 채집, 낚시, 채광, 장사 | 직업별 드랍·칭호 스테이지 |
| **가방** | 아이템 목록, 카테고리별 탭 | 농부/낚시꾼/광부/장사꾼/계약서/상자/소비 |
| **계약서** | 일반계약서 구매(1000G), 조합(10→1) | 고급→전문→명인→마스터 |
| **장터** | 3섹션 | 일반판매, 상인조합 풀, 상인판매 등록·구매 |
| **광장** | 철수 vs 영희 배팅 | 30분마다 경기, 30~100G 배팅 |
| **전투** | 하루 3회, 먹이사슬 크리 | 승/패/무승부, 골드 10% |
| **강화** | 전투능력 강화 | 강화석·고급강화석 조합 |
| **크로스조합** | 직업 간 레어+일반 조합 | 슈퍼푸드, 이직계약서, 강화석 조각 |
| **소비** | 에너지드링크, 슈퍼푸드 | 에너지 회복 |
| **상자** | 로또상자, 나락상자 | 일일 제한 |
| **채팅** | WebSocket 실시간 채팅 | 채널별 |

### 2.2 데이터 구조 (핵심)

- **유저**: `accountId`, `characterName`(아이디와 동일), `characterType`(직업), `level`, `exp`, `gold`, `energy`, `maxEnergy`, `combatPower`, `inventory`, 직업별 `harvestCount`/`fishCount`/`miningCount`, 칭호 관련 `jobTitle*`, `lastResetDate`, `battlesToday`
- **장터**: `guildPool`(상인조합 풀), `merchantListings`(상인판매 등록)

### 2.3 파일 구조

```
public/
  index.html     # 화면 (캐릭터생성/채널/지도/섹터/모달)

  js/app.js      # API 호출, UI 이벤트

  server.js      # HTTP + WebSocket, 정적 파일
  webApi.js      # /api/state, /api/command, /api/plaza/round
  chatServer.js  # attachChat (채팅)
  game/
    gameEngine.js   # 핵심 로직 (handleGameCommand, buildState)
    battleSystem.js # 먹이사슬, 전투 판정
    plazaGamble.js # 광장 경기·배팅
docs/
  게임_계획서.md   # 이 문서
  가이드서.md     # 플레이어 가이드
  게임_디자인_문서.md
  평민_마스터_육성_계획.md

```
